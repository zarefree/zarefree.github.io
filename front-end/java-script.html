JavaScript ​

各 for 循环性能对比及原理 ​

  • for...of 循环
js
for (const element of object) {
}

优先使用级最高

  • forEach 循环
js
array.forEach((element) => {});

需遍历整个数组、无需终止的情况下使用

  • 普通 for 循环
js
for (let i = 0; i < 5; i++) {
  console.log(i);
}

性能最好，但可读性比较差

  • for...in 循环
js
for (const key in object) {
  if (Object.prototype.hasOwnProperty.call(object, key)) {
    const element = object[key];
  }
}

非必要的情况下不建议使用

includes 和 indexOf ​

  • includes
js
const arr = [1, 2, 3, NaN];
console.log(arr.includes(2)); // true
console.log(arr.includes(NaN)); // true

能正确处理 NaN，推荐优先使用

  • indexOf
js
const arr = [1, 2, 3, NaN];
console.log(arr.indexOf(2)); // 1
console.log(arr.indexOf(NaN)); // -1

不能正确处理 NaN，在需要获取坐标的情况下使用

splice 和 slice ​

splice ​

语法：splice(start, deleteCount, ...items)

  • 删除数组里的某个元素
js
const list = [1, 2, 3];
list.splice(2, 1);
console.log(list); // [ 1, 2 ]
  • 替换数组里的某个元素
js
const list = [1, 2, 3];
list.splice(2, 1, 4);
console.log(list); // [ 1, 2, 4 ]
  • 向数组的某个位置插入元素
js
const list = [1, 2, 3];
list.splice(3, 0, 4, 5);
console.log(list); // [ 1, 2, 3, 4, 5 ]

slice ​

语法：slice(start, end)

  • 返回数组某个坐标到末尾的元素
js
const list = [1, 2, 3, 4, 5];
console.log(list.slice(2)); // [ 3, 4, 5 ]
  • 返回数组的某些元素
js
const list = [1, 2, 3, 4, 5];
console.log(list.slice(1, 3)); // [ 2, 3 ]
